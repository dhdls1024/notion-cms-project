# Minimalist Link-in-Bio (Notion CMS Hub) MVP PRD

## 핵심 정보

**목적**: Notion 데이터베이스를 CMS로 활용하여, 코드 배포 없이 Notion 수정만으로 실시간 업데이트되는 개인 링크 모음(디지털 명함) 사이트를 제공한다.
**사용자**: Notion을 사용하며 개발 없이 링크 페이지를 관리하고 싶은 1인 개발자 또는 크리에이터.

---

## 사용자 여정

```
1. [홈 - 링크 허브 페이지] 진입
   ↓ 페이지 로드 시 Notion API 자동 호출

2. Notion 데이터 로딩
   ↓ 서버 컴포넌트에서 fetch → 링크 목록 렌더링

   로딩 중  → Skeleton UI 표시
   로드 완료 → 카테고리 탭 + 링크 카드 목록 표시
   ↓

3. [카테고리 탭 선택]
   ↓ 탭 클릭 → 해당 카테고리 링크만 필터링 표시

4. [링크 카드 인터랙션]
   ↓ 사용자 선택 분기

   링크 클릭     → 외부 URL 새 탭으로 열기
   복사 버튼 클릭 → 클립보드 복사 + sonner 토스트 알림
   ↓

5. [다크모드 전환]
   헤더 토글 클릭 → 시스템/라이트/다크 모드 순환
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | Notion 데이터 동기화 | Notion API로 링크 데이터(제목, URL, 아이콘, 카테고리, 활성화 여부) 가져오기 | 코드 수정 없이 콘텐츠 관리 — 핵심 가치 제공 | 홈 페이지 |
| **F002** | 반응형 링크 카드 렌더링 | 모바일 우선 디자인, 리스트/그리드 레이아웃으로 링크 카드 표시 | 링크 허브의 핵심 UI — 없으면 서비스 불가 | 홈 페이지 |
| **F003** | 다크모드 전환 | next-themes로 시스템 감지 및 수동 라이트/다크 전환 | 가독성과 사용자 경험 — 개인 브랜딩 사이트 필수 요소 | 홈 페이지 |
| **F004** | 링크 URL 복사 | 복사 버튼 클릭 시 클립보드 복사 + sonner 토스트 알림 | 모바일에서 URL 공유의 핵심 편의 기능 | 홈 페이지 |
| **F005** | 카테고리 필터링 | 카테고리별 탭/필터 UI로 링크 분류 표시 | 링크가 많을수록 필수인 탐색 기능 | 홈 페이지 |

### 2. MVP 이후 기능 (제외)

- 관리자 대시보드 (Notion이 CMS 역할을 대체)
- 링크 클릭 통계 및 분석
- 커스텀 도메인 연결 UI
- 소셜 공유 버튼
- 애니메이션 및 테마 커스터마이징

## 페이지별 상세 기능

### 홈 페이지 (링크 허브)

> **구현 기능:** `F001`, `F002`, `F003`, `F004`, `F005` | **인증:** 불필요 (공개 페이지)

| 항목 | 내용 |
|------|------|
| **역할** | 서비스의 유일한 페이지. Notion DB에서 링크를 가져와 카테고리 필터와 함께 반응형 카드로 렌더링하는 디지털 명함 허브 |
| **진입 경로** | 사이트 루트 접속 시 자동 진입. 별도 내비게이션 없음 |
| **사용자 행동** | 링크 목록 탐색 → 카테고리 탭으로 필터링 → 링크 클릭(외부 이동) 또는 복사 버튼 클릭(클립보드 저장) → 다크모드 토글로 테마 전환 |
| **주요 기능** | - Notion API fetch로 활성화(`Active=true`) 링크만 로드 (F001)<br>- 로딩 중 Skeleton 카드 표시, 에러 시 Fallback UI (F001)<br>- 카테고리 탭 클릭 시 클라이언트 사이드 필터링 (F005)<br>- 모바일 1열 / 태블릿 2열 / 데스크탑 3열 그리드 카드 레이아웃 (F002)<br>- 각 카드의 복사 버튼 클릭 → `navigator.clipboard.writeText` + sonner `toast.success` 알림 (F004)<br>- 헤더 우측 `ThemeToggle` 버튼으로 라이트/다크 전환 (F003)<br>- **외부 링크 열기** 버튼 (새 탭 `target="_blank"`) |
| **다음 이동** | 외부 링크 클릭 → 새 탭으로 외부 URL 열기 (현재 페이지 유지), 복사 → 현재 페이지 유지 + 토스트 표시 |

## 데이터 모델

### Notion 데이터베이스 속성 구조

이 프로젝트는 별도 DB 없이 **Notion 데이터베이스를 단일 데이터 소스**로 사용한다.

#### Links 데이터베이스 (Notion)

| 속성명(Notion) | 속성 타입 | 설명 |
|--------------|-----------|------|
| Name | Title | 링크 표시 제목 (예: "GitHub", "블로그") |
| URL | URL | 연결할 외부 주소 |
| Icon | Files & media 또는 Text | 링크 아이콘 (이모지 또는 이미지 URL) |
| Category | Select | 카테고리 분류 (예: "개발", "SNS", "포트폴리오") |
| Active | Checkbox | 체크 시 공개, 미체크 시 숨김 |
| Order | Number | 카드 표시 순서 (오름차순 정렬) |

#### 앱 내부 TypeScript 타입 (types/index.ts)

```ts
// Notion API 응답을 앱에서 사용하는 형태로 정규화한 타입
type LinkItem = {
  id: string         // Notion 페이지 ID
  title: string      // Name 속성
  url: string        // URL 속성
  icon: string       // Icon 속성 (이모지 또는 이미지 URL)
  category: string   // Category 속성
  active: boolean    // Active 속성
  order: number      // Order 속성
}
```

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16** (App Router) - 서버 컴포넌트로 Notion API fetch, SEO 최적화
- **React 19** - UI 라이브러리 (동시성 기능, use() 훅 활용 가능)
- **TypeScript 5** - 타입 안전성 (Notion API 응답 타입 정의 포함)

### 스타일링 & UI

- **Tailwind CSS 4** - PostCSS 플러그인 방식 (`@import "tailwindcss"`), 설정 파일 불필요
- **shadcn/ui** - 카드, 탭, 버튼, 스켈레톤 컴포넌트 활용
- **Lucide React** - 복사, 링크, 다크모드 아이콘

### 테마 & 알림

- **next-themes** - `ThemeProvider`로 시스템 감지 + 수동 다크모드 전환 (`attribute="class"`)
- **sonner** - 링크 복사 성공/실패 토스트 알림 (`<Toaster richColors />`)

### 데이터 연동

- **@notionhq/client** - Notion 공식 Node.js SDK, 데이터베이스 쿼리 및 응답 파싱
- **Next.js 서버 컴포넌트 fetch** - 빌드 시 또는 ISR로 Notion 데이터 가져오기

### 유틸리티

- **clsx + tailwind-merge** - `cn()` 유틸리티로 조건부 클래스 병합 (`lib/utils.ts`)
- **react-hook-form + zod** - 향후 관리 기능 추가 시 폼 검증용 (현재 MVP에서는 미사용)

### 배포 & 호스팅

- **Vercel** - Next.js 16 최적화 배포 플랫폼, 환경 변수로 Notion API 키 관리

### 환경 변수 (.env.local)

```
NOTION_API_KEY=secret_xxxx          # Notion Integration 토큰
NOTION_DATABASE_ID=xxxx             # Links 데이터베이스 ID
```
